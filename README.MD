# LiteraryMatch: Personalized Book Recommender System

LiteraryMatch is an end-to-end web application designed to help users discover new books. By utilizing natural language processing (NLP) to analyze book descriptions, the system delivers personalized book recommendations. The platform features user authentication, an interactive search mechanism, and a curated landing page displaying top-rated books.

---

## Key Features

* **Content-Based Recommendation Engine:** Employs TF-IDF (Term Frequency-Inverse Document Frequency) Vectorization and Cosine Similarity on book descriptions to find and suggest closely related reads.
* **Curated Homepage ("Top 50 Books"):** Automatically displays 50 highly rated books on the main dashboard for immediate discovery.
* **Interactive Search:** Allows users to find specific titles within the catalog through case-insensitive text search.
* **User Authentication:** Secure signup, login, and session-based route protection implemented with Flask sessions and hashed passwords.
* **Responsive UI:** A clean, responsive interface styled with Tailwind CSS, supporting seamless navigation across desktop and mobile screens.

## Architecture & How It Works

### The Recommendation Pipeline
1. **Data Ingestion (ETL):** A local dataset of raw books is parsed, cleansed, and stored in MongoDB using a custom preprocessing pipeline.
2. **Data Fetching:** When a user requests a recommendation for a specific book, the application retrieves the entire book catalog from the MongoDB database as a Pandas DataFrame.
3. **Text Vectorization:** A `TfidfVectorizer` processes the `description` column, removing common English stop words and converting the text into numerical feature vectors representing word importance.
4. **Similarity Calculation:** The system calculates a similarity score matrix using `cosine_similarity` on the TF-IDF vectors.
5. **Ranking & Filtering:** It identifies the target book, ranks all other books based on their similarity score, and returns the top 5 matches to display to the user.

## Getting Started

### Prerequisites

* **Python 3.8+**
* **MongoDB** (running locally on port `27017`)
* **Goodreads Dataset:** Downloaded `books_enriched.csv` file.

### Installation & Setup

1. **Clone the repository:**
   ```bash
   git clone https://github.com/AnkitXAI/literary-match.git
   cd literary-match
   ```

2. **Prepare the Dataset:**
   Place your downloaded `books_enriched.csv` file directly into the root folder of the project (`literary-match/`).

3. **Create and activate a virtual environment:**
   ```bash
   python -m venv venv
   # On macOS/Linux:
   source venv/bin/activate
   # On Windows:
   venv\Scripts\activate
   ```

4. **Install the dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

5. **Run the ETL Pipeline:**
   Ensure your local MongoDB instance is active. Execute the script to process your local dataset and load it into your database:
   ```bash
   python ingest_data.py
   ```

6. **Run the application:**
   ```bash
   python app.py
   ```

7. **Access the application:**
   Open your browser and navigate to `http://127.0.0.1:5000/`.
